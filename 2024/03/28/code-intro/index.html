C++ 001 -> copy函数详解

这是复制

int main()
{
	int a = 5;
	int b = a;
}

这是指针的复制

int main()
{
	int* a = new int(5);
	int* b = a;
}

这种复制不会改变被复制的值

现在我们想要写一个字符串类，看看我们怎么才能使他具有可复制性以及面临的挑战

#include <iostream>
#include <string>

class String
{
private:
	char* m_Buffer;
	unsigned int m_Size;
public:
	String(const char* string)
	{
		m_Size = strlen(string);
		m_Buffer = new char[m_Size + 1];
		memcpy(m_Buffer, string, m_Size);
		
	}
	friend std::ostream& operator<<(std::ostream& stream, const String& string);

};

std::ostream& operator<<(std::ostream& stream, const String& string)
{
	stream << string.m_Buffer;
	return stream;
}

int main()
{
	String string = "Hello";
	std::cout << string << std::endl;
	std::cin.get();
}

#include <iostream>
#include <string>

class String
{
private:
	char* m_Buffer;
	unsigned int m_Size;
public:
	String(const char* string)
	{
		m_Size = strlen(string);
		m_Buffer = new char[m_Size + 1];
		memcpy(m_Buffer, string, m_Size);
		
	}
	friend std::ostream& operator<<(std::ostream& stream, const String& string);


	~String()
	{
		delete[] m_Buffer;
	}
};

std::ostream& operator<<(std::ostream& stream, const String& string)
{
	stream << string.m_Buffer;
	return stream;
}

int main()
{
	String string = "Hello";
	String second = string;

	std::cout << string << std::endl;
	std::cout << second << std::endl;

	std::cin.get();
}

我们进行了对我们自己定义的String类成员进行了复制。内存中现在有了两个String，有相同的内存地址，程序会崩溃，这会调用delete两遍。

即使我们再写一个类，改变类的字符，但是仍会崩溃，因为指针还是一样的。

我们需要deep copying，方法就是写copy构造函数。

在public中加上

String(const String& other);

这样就默认提供了构造函数（本质是调用了memcpy函数）

如果我们不允许复制：

String(const String& other) = delete;

假设有两个对象，s1，s2.其中String s2 = “Hello”,此时s2已被构造，String s1 = s2，此时s1未被构造，所以会先调用拷贝构造函数，但是由于拷贝函数被delete，所以编译器自此报错。使用s1=s2，也会正常编译，因为s1和s2已被构造，会优先调用重写的operator“=”。因为s1尚未创建所以重写的operator=方法不会被调用，String s1 = “123”，此时即使拷贝构造被delete，也会报错。

所以在调用的时候，我们给参数传上const的参数，运用上&就行了。
